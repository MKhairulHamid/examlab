Overriding padding/size: The className prop is appended after the variant classes, but Tailwind utilities at the same specificity don't always win by order alone. Use the ! important prefix (!px-10, !py-4) to guarantee your overrides take effect. This is the correct pattern — not a hack.

Modal + ModalHeader

src/design-system/Modal.jsx — accessible overlay with body-scroll lock, backdrop blur, and slide-up animation.

Preview

How it renders
Welcome Back
Login to access exam practice tests
Form content goes here — LoginForm, SignupForm, plan cards, etc.

Behaviour

  • Body scroll lock: sets document.body.style.overflow = 'hidden' when open, restores on close or unmount
  • Backdrop click to close: clicking outside the modal panel fires onClose
  • Animation: animate-[slideUp_0.2s_ease] defined in tailwind.config.js
  • Scroll within modal: max-h-[90vh] overflow-y-auto — long content scrolls inside the panel, not the page
  • Backdrop: rgba(0,0,0,0.5) + backdrop-filter: blur(4px) (inline style — CSS variable not available here)

PropTypeNotes

Usage

import { Modal, ModalHeader } from '../design-system'

function AuthModal({ isOpen, onClose }) {
  return (
    <Modal isOpen={isOpen} onClose={onClose} maxWidth="max-w-md">
      <ModalHeader
        title="Welcome Back"
        description="Login to access exam practice tests"
        onClose={onClose}
      />
      <div className="px-6 py-5">
        <LoginForm onSuccess={onClose} />
      </div>
    </Modal>
  )
}
Don't add your own close button when using ModalHeader — it already renders the ✕ icon. If you build a custom header without ModalHeader, add your own close button and call onClose from it.

Using Modal without ModalHeader

// For modals that need a fully custom layout
<Modal isOpen={show} onClose={handleClose} maxWidth="max-w-lg">
  <div className="p-8 text-center">
    <div className="text-5xl mb-4">🎉</div>
    <h2 className="text-xl font-bold mb-2">Payment Successful</h2>
    <Button variant="primary" onClick={handleClose}>Continue</Button>
  </div>
</Modal>

Guidelines

Inline Style Rules

When to use Tailwind utilities vs style={{}} vs CSS class files.

The hierarchy

ApproachWhen to use
Design system componentAny time a Button, Card, Modal, or Container would fit. Always first choice.
Tailwind utility classesStructural layout, spacing, typography, static colors. Everything that isn't a dynamic runtime value.
CSS class (index.css)Complex multi-property component styles (e.g. .exam-interface, .question-set-card) that are used in many places or need pseudo-element selectors.
Inline style={{}}Only for values that depend on a runtime JS variable. No exceptions.

Inline style — allowed cases

// ✅ Dynamic color from runtime data
<div style={{ borderLeft: `4px solid ${passColor}` }}>

// ✅ Dynamic width from computed state
<div style={{ width: `${(timeLeft / duration) * 100}%` }}>

// ✅ Dynamic background color with ternary
<div style={{
  backgroundColor: timeLeft < duration * 0.1 ? '#ef4444' : '#00D4AA'
}}>

// ✅ Conditional opacity from boolean
<div style={{ opacity: isLocked ? 0.7 : 1 }}>

Inline style — not allowed

// ❌ Static value — use Tailwind instead
<div style={{ textAlign: 'center' }}>          // → className="text-center"
<div style={{ marginBottom: '1rem' }}>          // → className="mb-4"
<div style={{ display: 'flex', gap: '1rem' }}>  // → className="flex gap-4"
<div style={{ fontSize: '4rem' }}>              // → className="text-[4rem]"

// ❌ Hover handler for static color — use Tailwind hover:
onMouseEnter={e => e.currentTarget.style.background = '#fff'}  // → className="hover:bg-white"

// ❌ Button with all inline styles — use <Button> or Tailwind
<button style={{ padding: '...', background: '...', borderRadius: '...' }}>

Hover states with runtime colors

Tailwind's hover: variant only accepts static class names, not runtime values. If the hover color depends on JS data, keep the event handler:

// ✅ Correct — p.color is a runtime variable per card
<div
  onMouseEnter={e => e.currentTarget.style.borderColor = p.color}
  onMouseLeave={e => e.currentTarget.style.borderColor = 'transparent'}
>

// ✅ Also correct — static hover with Tailwind
<div className="hover:bg-white/20 hover:border-white/40 transition-colors">

Tailwind ! important prefix

When the design system component sets utility classes internally and you need to override them from outside, use the ! prefix. This is the sanctioned Tailwind way — not a hack.

// Button has built-in px-6 py-3 for lg size.
// To make a larger hero button, override with !important:
<Button variant="primary" className="!px-10 !py-4 !text-[1.0625rem]">
  Start Free Today
</Button>
