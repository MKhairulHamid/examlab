AWS Certified Generative AI Developer – Professional (AIP-C01)

Everything we need to know about this exam — structure, domains, candidate prep strategy, and simulation design.

What This Exam Is

The AWS Certified Generative AI Developer – Professional (AIP-C01) validates the ability to integrate foundation models (FMs) into applications and business workflows and run them in production on AWS. It is AWS's most advanced AI engineering credential — the engineering counterpart to the foundational AI Practitioner (AIF-C01). Where AIF tests AI literacy, AIP tests production GenAI build skill: RAG, agents, safety, governance, cost, and operations.

  • Target candidate: A GenAI developer with 2+ years building production applications on AWS or open source, general AI/ML or data-engineering experience, and 1+ year hands-on implementing GenAI solutions.
  • Validated abilities: Design RAG/vector-store/knowledge-base architectures; integrate FMs into apps and workflows; apply prompt engineering and management; build agentic AI; optimize for cost, performance, and business value; implement security, governance, and responsible AI; troubleshoot, monitor, and evaluate FMs.
  • Out of scope for the candidate: Model development and training, advanced ML techniques, and data/feature engineering — this is an integration and operations exam, not a model-training exam.

Out-of-Scope Tasks (Do NOT Test)

The official exam guide places these job tasks out of scope for the candidate. Questions should not require them:

  • Model development and training (building or pre-training foundation models)
  • Advanced ML techniques (custom architectures, novel training methods)
  • Data engineering and feature engineering as a discipline
Exam tip: AIP-C01 tests how to integrate, secure, operate, and optimize foundation models — not how to build them. If a question is really about training a model from scratch or designing a novel architecture, it is out of scope.

Domain 1 · 31%

Foundation Model Integration, Data Management & Compliance

The largest and most foundational domain. Designing GenAI solutions, choosing and customizing models, building data pipelines, and the full RAG stack — vector stores, retrieval, and prompt governance.

Task 1.1 — Analyze requirements and design GenAI solutions

Skills in:

  • Creating architectural designs aligned to business needs and constraints (FM choice, integration patterns, deployment strategies)
  • Building proof-of-concept implementations on Amazon Bedrock to validate feasibility, performance, and value before full build-out
  • Standardizing components with the AWS Well-Architected Framework and its Generative AI Lens

Task 1.2 — Select and configure FMs

Skills in:

  • Choosing FMs on benchmarks, capability, and limitation analysis for the use case
  • Designing abstraction patterns for dynamic model selection/provider switching without code changes (Lambda, API Gateway, AppConfig)
  • Building resilient systems (Bedrock Cross-Region Inference, circuit breakers, graceful degradation)
  • Deploying customized FMs and managing lifecycle (SageMaker AI, LoRA/adapters, Model Registry, rollback)

Task 1.3 — Implement data validation and processing pipelines for FM consumption

Skills in:

  • Building data validation workflows (AWS Glue Data Quality, SageMaker Data Wrangler, Lambda, CloudWatch metrics)
  • Processing multimodal data — text, image, audio, tabular (Bedrock multimodal, SageMaker Processing, Amazon Transcribe)
  • Formatting input to model-specific requirements (JSON for Bedrock, structured payloads for SageMaker endpoints, conversation formatting)
  • Enhancing input quality (Bedrock reformatting, Comprehend entity extraction, Lambda normalization)

Task 1.4 — Design and implement vector store solutions

Skills in:

  • Designing vector database architectures (Bedrock Knowledge Bases, OpenSearch with Neural plugin, Aurora/RDS, DynamoDB)
  • Building metadata frameworks for precision and filtering (timestamps, authorship, domain tags, S3 metadata)
  • Scaling search (OpenSearch sharding, multi-index, hierarchical indexing)
  • Keeping stores current (incremental updates, change detection, scheduled refresh, synchronization)

Task 1.5 — Design retrieval mechanisms for FM augmentation

Skills in:

  • Document chunking (Bedrock chunking, fixed-size, hierarchical, semantic)
  • Selecting/configuring embedding models (Amazon Titan embeddings, dimensionality, domain fit, batch generation)
  • Deploying vector search (OpenSearch, Aurora pgvector, Bedrock Knowledge Bases)
  • Improving relevance (hybrid keyword+vector search, Bedrock reranker models)
  • Advanced query handling (expansion, decomposition, transformation) and consistent access (function calling, MCP)

Task 1.6 — Implement prompt engineering strategies and governance

Skills in:

  • Instruction frameworks controlling behavior (Bedrock Prompt Management, Guardrails, output templates)
  • Prompt management/governance (parameterized templates, versioning, approval workflows, S3 repositories, CloudTrail/CloudWatch auditing)
  • Prompt quality assurance and regression testing (Lambda, Step Functions, CloudWatch)
  • Advanced techniques (chain-of-thought, structured I/O, feedback loops) and Bedrock Prompt Flows for multi-step logic
Domain 2 · 26%

Implementation and Integration

The build domain. Agentic AI and tool use, model deployment strategies, enterprise and API integration, and the application patterns and developer tools that turn FMs into shipped features.

Task 2.1 — Implement agentic AI solutions and tool integrations

Skills in:

  • Building autonomous systems with memory/state (Strands Agents, AWS Agent Squad, MCP for agent-tool interactions)
  • Structured reasoning (ReAct, chain-of-thought via Step Functions) and safeguarded workflows (stopping conditions, timeouts, IAM boundaries, circuit breakers)
  • Multi-model coordination, human-in-the-loop review, and tool integration with error handling and parameter validation
  • Model extension via MCP servers (Lambda for lightweight, ECS for complex tools)

Task 2.2 — Implement model deployment strategies

Skills in:

  • Deploying FMs by need (Lambda on-demand, Bedrock provisioned throughput, SageMaker endpoints for hybrid)
  • Addressing LLM-specific deployment challenges (memory, GPU utilization, token throughput, container patterns)
  • Optimizing deployment (right-sized models, smaller task-specific models, API-based model cascading)

Task 2.3 — Design and implement enterprise integration architectures

Skills in:

  • Connecting FMs to existing systems (API integrations, event-driven loose coupling via EventBridge, data sync)
  • Secure access (identity federation, RBAC, least-privilege FM access)
  • Cross-environment compliance (Outposts, Wavelength) and CI/CD + GenAI gateway architectures (CodePipeline, CodeBuild, centralized abstraction, observability)

Task 2.4 — Implement FM API integrations

Skills in:

  • Flexible interaction (Bedrock APIs, SDKs + SQS for async, API Gateway with request validation)
  • Real-time interaction (Bedrock streaming APIs, WebSockets/SSE, chunked transfer encoding)
  • Resilience (exponential backoff, rate limiting, fallback, X-Ray observability) and intelligent model routing (static and Step Functions content-based)

Task 2.5 — Implement application integration patterns and development tools

Skills in:

  • FM API interfaces (API Gateway streaming, token-limit and retry handling)
  • Accessible interfaces (Amplify UI components, OpenAPI specs, Prompt Flows no-code builders)
  • Business system enhancements (Lambda, Step Functions document processing, Bedrock Data Automation)
  • Developer productivity and troubleshooting (Amazon Q Developer, CloudWatch Logs Insights, X-Ray)
Domain 3 · 20%

AI Safety, Security & Governance

Keeping GenAI safe, private, compliant, and responsible — input/output safety controls, data security and privacy, governance and audit, and responsible-AI principles.

Task 3.1 — Implement input and output safety controls

Skills in:

  • Content safety on inputs and outputs (Bedrock Guardrails, custom moderation workflows, toxicity detection)
  • Hallucination reduction (Knowledge Base grounding, confidence scoring, JSON Schema structured outputs)
  • Defense-in-depth (Comprehend pre-processing, model guardrails, Lambda post-processing, API Gateway filtering)
  • Threat detection (prompt-injection and jailbreak detection, input sanitization, safety classifiers, adversarial testing)

Task 3.2 — Implement data security and privacy controls

Skills in:

  • Protected environments (VPC endpoints, IAM secure-access patterns, Lake Formation, CloudWatch monitoring)
  • Privacy preservation (Comprehend & Macie PII detection, Bedrock data-privacy features, Guardrails output filtering, S3 retention)
  • Privacy-focused systems (data masking, anonymization while maintaining utility)

Task 3.3 — Implement AI governance and compliance mechanisms

Skills in:

  • Compliance frameworks (SageMaker model cards, Glue data lineage, metadata tagging, decision logs)
  • Data source tracking and traceability (Glue Data Catalog, source attribution, CloudTrail audit logging)
  • Organizational governance and continuous monitoring (misuse/drift/policy-violation detection, bias-drift monitoring, token-level redaction, output policy filters)

Task 3.4 — Implement responsible AI principles

Skills in:

  • Transparency (reasoning displays, confidence metrics, evidence/source attribution, Bedrock agent tracing)
  • Fairness evaluation (CloudWatch fairness metrics, A/B testing via Prompt Management/Flows, LLM-as-a-judge model evaluations)
  • Policy-compliant systems (Guardrails by policy, model cards documenting limitations, Lambda automated compliance checks)
Domain 4 · 12%

Operational Efficiency & Optimization for GenAI Applications

Making GenAI affordable and fast at scale — token and cost efficiency, application performance and latency, and GenAI-specific monitoring and observability.

Task 4.1 — Implement cost optimization and resource efficiency strategies

Skills in:

  • Token efficiency (estimation/tracking, context-window optimization, prompt compression, response limiting)
  • Cost-effective model selection (cost-capability tradeoff, tiered usage by complexity, price-to-performance)
  • Throughput (batching, capacity planning, auto-scaling, provisioned-throughput optimization)
  • Intelligent caching (semantic caching, prompt caching, deterministic request hashing, edge caching)

Task 4.2 — Optimize application performance

Skills in:

  • Latency-cost tradeoffs (pre-computation, latency-optimized Bedrock models, parallel requests, response streaming)
  • Retrieval performance (index optimization, query preprocessing, hybrid search with custom scoring)
  • Throughput and parameter tuning (batch inference, concurrency, temperature and top-k/top-p selection, A/B testing)

Task 4.3 — Implement monitoring systems for GenAI applications

Skills in:

  • Holistic observability (operational metrics, FM interaction tracing, business-impact dashboards)
  • GenAI-specific monitoring (CloudWatch token usage, hallucination/response-quality tracking, Bedrock Model Invocation Logs, cost anomaly detection)
  • Tool and vector-store operational monitoring, and FM-specific troubleshooting frameworks (golden datasets, output diffing, reasoning-path tracing)
Domain 5 · 11%

Testing, Validation & Troubleshooting

Proving and repairing GenAI quality — evaluation frameworks beyond traditional ML metrics, and systematic troubleshooting of FM applications.

Task 5.1 — Implement evaluation systems for GenAI

Skills in:

  • Quality assessment beyond traditional ML (relevance, factual accuracy, consistency, fluency)
  • Systematic evaluation (Bedrock Model Evaluations, A/B and canary testing, cost-performance analysis)
  • User-centered and QA processes (feedback/rating interfaces, regression testing, automated quality gates)
  • RAG evaluation, LLM-as-a-Judge, retrieval-quality testing, agent-performance frameworks (Bedrock Agent evaluations), deployment validation

Task 5.2 — Troubleshoot GenAI applications

Skills in:

  • Content-handling issues (context-window overflow, dynamic chunking, truncation analysis)
  • FM integration issues (error logging, request validation, response analysis)
  • Prompt troubleshooting (testing frameworks, version comparison, systematic refinement)
  • Retrieval issues (relevance analysis, embedding-quality diagnostics, drift monitoring, chunking remediation) and prompt-maintenance observability (CloudWatch Logs, X-Ray, schema validation)
Candidate Prep

How We Help AIP-C01 Candidates

Who takes this exam, where they struggle, and how to build a study path that gets them to 750+.

Who Takes This Exam

  • GenAI / AI engineers already building with Amazon Bedrock who want to validate production-grade skill.
  • Developers and solutions architects moving from "calling a model API" to designing RAG, agents, and governed AI platforms.
  • ML engineers shifting from training models to integrating foundation models into applications.

What Candidates Struggle With

  • RAG vs. fine-tuning — knowing that grounding in private, changing data with citations is RAG, while fine-tuning teaches style/behavior, not fresh facts.
  • The full RAG stack — separating the storage half (vector store, metadata, sync) from the retrieval half (chunking, embeddings, hybrid search, reranking) and knowing which lever fixes which symptom.
  • Agents vs. Prompt Flows vs. a single prompt — when a task needs autonomous tool use, when deterministic branching suffices, and when better prompting is enough.
  • Guardrails vs. prompt instructions — that safety policy must be enforced at the platform level, independent of (and resilient to) prompt changes and prompt injection.
  • GenAI-specific operations — token cost, semantic/prompt caching, latency-cost tradeoffs, and monitoring metrics (hallucination rate, token bursts) that have no traditional-ML equivalent.
  • GenAI-specific evaluation — LLM-as-a-Judge, RAG evaluation, and golden datasets rather than accuracy/F1 alone.

Recommended Study Path

  1. Domain 1 — FM integration & the RAG stack first. Highest weight (31%) and the foundation everything else builds on. Master archetype selection, the vector-store/retrieval split, and prompt governance.
  2. Domain 2 — Implementation & integration. Agents, deployment, API patterns, and enterprise integration. Builds directly on D1's foundations.
  3. Domain 3 — Safety, security & governance. Guardrails, PII protection, prompt-injection defense, and responsible AI — heavily interconnected with D1 and D2.
  4. Domains 4 & 5 — Operations and evaluation last. Smaller (12% + 11%) and best learned once the build skills are solid: cost/performance/monitoring, then evaluation and troubleshooting.

AI Explanation Quality for This Exam

AIP-C01 questions are scenario-based and often have two plausible options. Every explanation must say WHY the chosen answer wins on the specific requirement and WHY each distractor falls short. Common patterns to cover:

  • For RAG-vs-fine-tuning questions: explain that the requirement (current facts, citations, private data) points to RAG, and that fine-tuning teaches behavior not fresh facts.
  • For retrieval-symptom questions: tie the symptom to the lever — "misses exact terms" → hybrid search; "related but not best-ranked" → reranking; "context cut off" → chunking.
  • For safety questions: explain that platform Guardrails enforce policy regardless of prompt, where prompt instructions can be bypassed by injection.
  • For cost/latency questions: explain which technique each option optimizes (semantic caching avoids invocations; smaller/tiered models cut token cost; streaming improves perceived latency).

Question Writing Rules (AIP-C01 specific)

  • Scenario-based stems — "A company is building a RAG assistant that…" Never "What is an embedding?"
  • Integration and operations, not model training — never ask candidates to design a novel architecture or pre-train a model.
  • Make the requirement the crux — the deciding detail (citations needed, data changes hourly, must isolate tenants, lowest token cost) should select the answer.
  • Symptom → lever framing — many D1/D5 questions reduce to matching a retrieval/prompt symptom to the correct fix.
  • Two-plausible-options for Sim 3 — comparative questions where both options work but one fits the stated constraint better.

Distractor Quality Guidelines

  • Use real GenAI confusions: RAG vs. fine-tuning; agents vs. Prompt Flows; Guardrails vs. prompt instructions; hybrid search vs. reranking vs. chunking.
  • For safety questions: include a distractor that puts the rule in the prompt instead of platform Guardrails (plausible but bypassable).
  • For cost questions: make each distractor optimize a real but wrong dimension (bigger model, more retries, no caching).
  • Every question carries a full ai_cache explanation payload — overview, per-option reasons, concept guide, and official links.
